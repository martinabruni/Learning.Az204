# API Management Subscriptions and Keys

## Definition

A subscription in [[azure-api-management]] is a named container for a **pair of subscription keys** (primary + secondary). Subscriptions control access to APIs and provide granular control over permissions and policies.

## Subscription Key Mechanics

- Keys are **unique autogenerated strings**.
- Passed in HTTP requests via:
  - **Header**: `Ocp-Apim-Subscription-Key: <key>`
  - **Query string**: `?subscription-key=<key>`
- Missing or invalid key → `401 Access Denied` from the [[apim-gateway|gateway]].

## Dual Key Design

Each subscription has two keys to enable zero-downtime rotation:

1. Switch applications to use the **secondary** key.
2. Regenerate the **primary** key.
3. Switch applications back to the new primary key.

Keys can also be regenerated immediately if a key is suspected compromised.

## Subscription Scopes

| Scope | Applies To |
|-------|-----------|
| All APIs | Every API accessible from the gateway |
| Single API | One imported API and all its endpoints |
| Product | All APIs in a product (with that product's rules and quotas) |

## Developer Workflow

1. Developer discovers API in the Developer Portal.
2. Developer submits a subscription request for a product.
3. Publisher approves (or auto-approval fires if configured).
4. Publisher sends the subscription key securely.
5. Developer includes the key in every API call.

## Alternative Security Mechanisms

API Management also supports:
- **OAuth 2.0**
- **Client certificates** (TLS mutual auth — see [[apim-certificates]])
- **IP allow listing**
