# Secure APIs by Using Subscriptions

> [!source] learn.microsoft.com_en-us_training_modules_explore-api-management_6-secure-access-api-subscriptions.pdf

## Overview

Subscription keys are the most common mechanism to secure access to APIs published through [[azure-api-management]]. Developers include a valid subscription key in every HTTP request; the [[apim-gateway|gateway]] rejects calls without one.

> [!source] learn.microsoft.com_en-us_training_modules_explore-api-management_6-secure-access-api-subscriptions.pdf
> "When you publish APIs through API Management, it's easy and common to secure access to those APIs by using subscription keys."

## Key Claims

- The gateway rejects calls without a subscription key; they are never forwarded to backend services.
- A subscription is a named container for a **pair** of subscription keys (primary + secondary).
- Developers can get subscriptions without publisher approval (auto-approval is configurable).
- API publishers can also create subscriptions directly for API consumers.
- API Management also supports: **OAuth 2.0**, **client certificates**, and **IP allow listing**.

## Subscriptions and Keys

### Subscription Key Mechanics

- A subscription key is a **unique autogenerated key**.
- Can be passed in:
  - **HTTP header**: default header name is `Ocp-Apim-Subscription-Key`
  - **URL query string**: default parameter name is `subscription-key`
- Missing key returns `401 Access Denied` from the gateway.

### Dual Key Design

Every subscription has two keys (primary and secondary):
- Enables zero-downtime key rotation: switch apps to secondary key, then regenerate primary.
- Keys can be regenerated at any time (e.g., if a key is compromised).

### Subscription Scopes

| Scope | Description |
|-------|-------------|
| All APIs | Applies to every API accessible from the gateway |
| Single API | Applies to one imported API and all its endpoints |
| Product | Applies to all APIs in a product; products can have different access rules, quotas, and terms |

## Calling an API with a Subscription Key

Pass via header:
```bash
curl --header "Ocp-Apim-Subscription-Key: <key string>" https://<apim-gateway>.azure-api.net/api/path
```

Pass via query string:
```bash
curl https://<apim-gateway>.azure-api.net/api/path?subscription-key=<key string>
```

## Workflow

1. Developer submits a subscription request.
2. Publisher approves (or auto-approval fires).
3. Publisher sends the subscription key securely (e.g., via encrypted message).
4. Developer includes the key in all API calls.
